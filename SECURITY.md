# Security Policy

## Supported Versions

Security fixes are provided only for the latest version available on the default branch.

| Version         | Supported |
| --------------- | --------- |
| Latest (`main`) | ✅ Yes     |
| Older versions  | ❌ No      |

## Reporting a Vulnerability

If you discover a security vulnerability, **please do not create a public GitHub issue**.

Instead, report it privately by contacting the repository owner with:

* A clear description of the vulnerability.
* Steps to reproduce the issue.
* The potential impact.
* Any proof of concept (if applicable).
* Suggested mitigations (optional).

Please allow reasonable time for the issue to be investigated and resolved before any public disclosure.

## Response Process

After receiving a report, the repository owner will:

1. Acknowledge receipt of the report.
2. Verify and assess the vulnerability.
3. Work on an appropriate fix.
4. Release a patch if necessary.
5. Credit the reporter where appropriate (unless anonymity is requested).

## Scope

This policy applies to:

* Application source code
* Authentication and authorization mechanisms
* API endpoints
* Database interactions
* File uploads and exports
* Third-party integrations
* Infrastructure and deployment configurations maintained within this repository

## Out of Scope

The following are generally considered out of scope:

* Vulnerabilities in third-party services or dependencies that are not caused by this project.
* Social engineering or phishing attempts.
* Denial-of-service attacks.
* Automated vulnerability scanner reports without a demonstrated, reproducible issue.
* Reports requiring physical access to systems.

## Responsible Disclosure

Please avoid publicly disclosing security issues until they have been reviewed and, where appropriate, addressed. Responsible disclosure helps protect users and maintain the security of the project.

Thank you for helping improve the security of Resume Solutions.
